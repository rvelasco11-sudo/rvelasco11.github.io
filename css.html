1. El Porcentaje (%): ¿Unidad o Factor? Existe un debate semántico sobre si el % es una unidad independiente. Técnicamente, en CSS, el porcentaje es un factor de escala. Comportamiento: Siempre hereda su valor de otra propiedad. Si lo usas en width, se refiere al ancho del elemento padre. Si lo usas en font-size, se refiere al tamaño de fuente del elemento padre. Diferencia clave: A diferencia de em o rem, el porcentaje es estrictamente relativo al contenedor inmediato (en dimensiones) o al valor heredado (en tipografía). 2. Diferencias Reales: em vs. rem vs. ex Esta es la parte donde muchos desarrolladores se confunden. La clave está en qué usan como base para el cálculo. rem (Root em) Es la unidad más segura y predecible. Referencia: Se basa exclusivamente en el tamaño de fuente del elemento raíz (). Cálculo: Por defecto, la mayoría de los navegadores tienen 1rem = 16px. Uso ideal: Para el layout general, márgenes y tamaños de fuente globales, ya que mantiene la consistencia en toda la página. em Es una unidad relativa al contexto local. Referencia: Se basa en el tamaño de fuente del propio elemento (o del padre si el elemento no tiene uno definido). El problema del "Efecto Cascada": Si tienes elementos anidados con em, el tamaño se multiplica. Ejemplo: Un div con 0.5em dentro de otro div de 0.5em resultará en un texto de un cuarto del tamaño original. Uso ideal: Para componentes específicos donde quieres que el padding o el margen escalen proporcionalmente al texto de ese componente (como un botón). ex Es una unidad poco común pero muy técnica y útil en tipografía avanzada. Referencia: Se basa en la altura de la letra "x" minúscula (la altura de x o x-height) de la fuente actual. Cálculo: No es fijo. Si cambias de tipografía (de Arial a Times New Roman), el valor de 1ex cambiará aunque el font-size sea el mismo, porque cada fuente tiene proporciones distintas. Uso ideal: Cuando necesitas que un elemento gráfico (como un icono o una línea) esté perfectamente alineado con la "panza" de las letras minúsculas.

El modelo de caja en CSS es el sistema que define a cada elemento de una web como una estructura rectangular compuesta por cuatro capas concéntricas: el contenido (texto o imágenes), el padding (espacio interno transparente), el borde (la línea que rodea al padding) y el margen (el espacio externo que separa la caja de otros elementos). La clave para entenderlo es la propiedad box-sizing, ya que por defecto (content-box) el padding y el borde se suman al ancho que tú defines, haciendo que la caja crezca más de lo esperado; en cambio, con border-box, el navegador ajusta el contenido para que el padding y el borde queden dentro del ancho total definido, logrando un diseño mucho más preciso y predecible.
